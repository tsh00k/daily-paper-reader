## 摘要
在机器人学习中，跨不同形态和环境的训练数据集规模化已成为学习通用机器人策略的主流范式。这些策略通常通过行为克隆从预先收集的演示中模仿动作进行训练。然而，由于机器人动作与数据采集机器人的动力学紧密相关，不同机器人可能需要不同的动作来实现相同的运动。这种不一致性阻碍了跨不同机器人的策略训练和部署。为解决这一问题，我们提出将笛卡尔状态增量作为跨机器人的通用动作表示，并引入状态预测与自适应命令执行（SPACE）框架。SPACE在三个层面处理机器人动力学变化：不同形态之间、同种形态的不同硬件单元之间，以及单个机器人在运行过程中的变化。它包含两个组件：（i）预测几何末端执行器位移的笛卡尔状态增量策略，以及（ii）将预测的笛卡尔状态增量转换为特定机器人控制命令的动作适配器。实验表明，当从不同形态和同种形态的不同硬件单元收集的数据中学习时，SPACE显著优于直接预测控制命令的策略。此外，SPACE在部署时的动力学变化（包括控制频率、物体重量和控制器增益的变化）下仍保持鲁棒性。项目页面可见于 http://haeone.site/space-website/。

## Abstract
In robot learning, scaling training datasets across diverse embodiments and environments has become a dominant paradigm for learning generalizable robot policies. These policies are commonly trained via behavior cloning to imitate actions from pre-collected demonstrations. However, since robot actions are tied to the dynamics of the data collection robot, different robots may require different actions to achieve the same motion. This discrepancy hinders both policy training and deployment across diverse robots. To address this, we propose using Cartesian state delta as a universal action representation across robots, and introduce State Prediction and Adaptive Command Execution (SPACE) framework. SPACE handles robot dynamics variation at three levels: across different embodiments, across hardware units of the same embodiment, and within a single robot during operation. It consists of two components: (i) a Cartesian state delta policy that predicts geometric end-effector displacement, and (ii) Action Adapter, which converts the predicted Cartesian state delta into robot-specific control commands. Experiments show that SPACE substantially outperforms policies that directly predict control commands when learning from data collected across different embodiments and across hardware units of the same embodiment. SPACE also remains robust under dynamics shifts at deployment, including changes in control frequency, object weight, and controller gains. ## 1. 核心问题与整体含义

- **研究动机**：在机器人学习（尤其行为克隆）中，不同机器人之间的动作表示与自身动力学紧密绑定：相同运动在不同机器人上需要不同的控制命令才能实现，且同一型号不同硬件单元甚至同一机器人在运行过程中也存在动力学差异。这导致跨机器人数据难以直接用于策略训练和部署，阻碍通用策略的发展。
- **核心挑战**：现有工作虽然使用不同的动作空间（关节/笛卡尔、绝对/增量），但都未从根本上解决控制命令与机器人动力学耦合的问题。跨形态数据集（如Open X-Embodiment）和多硬件数据集（如DROID）中，控制命令不一致严重损害学习效果。
- **整体含义**：本文提出通过预测**笛卡尔状态增量**（实际末端位移）作为通用动作表示，并借助**动作适配器**自适应地将该通用表示转换为特定机器人的控制命令，从而实现从异构数据中有效学习通用策略。

## 2. 方法论

### 2.1 核心思想

将策略的动作输出从“控制命令”（与机器人控制器相关）解耦为“实际运动”（与机器人动力学无关）。然后通过一个轻量级的在线自适应模块将通用运动转化为可执行命令。

### 2.2 关键技术细节

- **笛卡尔状态增量策略**：
  - 定义：在每个时间步 t，预测末端执行器从当前到下一时刻的位移 \(\Delta p_t = p_{t+1} - p_t\)，其中 \(p_t\) 包含位置和方向（用旋转矩阵转换）。
  - 优势：该量可直接从任何提供末端姿态的数据集中获取（无需控制命令）；不受控制器实现、机器人运动学和基坐标平移影响；可在不同形态和硬件单元间共享。
  - 策略输入：观测 \(o_t = \{p_t, I_t, l_t\}\)（末端位姿、图像、语言指令），输出 \(\Delta p_t\)。
  - 训练目标：\(\max_\theta \mathbb{E}_{\tau \sim D, (o_t, \Delta p_t) \sim \tau} [\log \pi_\theta(\Delta p_t | o_t)]\)。

- **动作适配器**：
  - **离线初始化**：收集少量随机轨迹（\(M=10\)，每段\(K=50\)步）构成校准数据集 \(D_{cal}\)，通过最小二乘拟合线性模型 \(\hat{u} = W_0 \Delta p + b_0\)，将通用位移映射为机器人的控制命令 \(u\)。
  - **在线自适应**：部署时使用LMS（最小均方）算法连续更新适配器参数。每个时间步：策略输出 \(\Delta p_t^{\text{target}}\)，适配器生成命令 \(\hat{u}_t = W_t \Delta p_t^{\text{target}} + b_t\)；执行后观测实际位移 \(\Delta p_t^{\text{obs}}\)，计算预测误差 \(e_t = W_t \Delta p_t^{\text{obs}} + b_t - \hat{u}_t\)；更新公式：\(W_{t+1} = W_t - \mu e_t (\Delta p_t^{\text{obs}})^\top\)，\(b_{t+1} = b_t - \mu e_t\)，学习率 \(\mu = 0.2\)。
  - 注：初始线性回归在整个实验中只需约1分钟（FR3机器人上），在线更新仅需一步梯度。

### 2.3 算法流程（文字说明）

1. 收集校准数据 \(D_{cal}\)（随机脚本策略执行10条轨迹，每段50步）；
2. 用线性回归初始化 \(W_0, b_0\)；
3. 每次 rollout 开始时重置适配器为初始值；
4. 循环每个时间步 t：
   - 策略预测 \(\Delta p_t^{\text{target}}\)；
   - 计算 \(\hat{u}_t = W_t \Delta p_t^{\text{target}} + b_t\)；
   - 执行命令 \(\hat{u}_t\)，观测新姿态 \(p_{t+1}\)，得到 \(\Delta p_t^{\text{obs}}\)；
   - 计算误差并更新 \(W_{t+1}, b_{t+1}\)。

## 6. 主要结论与发现

1. **跨形态学习**：SPACE在UR5+FR3共训的三个任务中平均比控制命令策略**成功率高30%**，在UR5→FR3零样本迁移中达到**92%成功率**（控制命令为0%）。在UMI+FR3场景中比控制命令策略**高50%**。
2. **跨硬件单元学习**：单硬件迁移（Robot1→Robot2）中，控制命令策略成功率从98%跌至18%，而SPACE仍达**84%**（仅下降12%）。在多硬件数据DROID训练中，SPACE在两个任务上分别比控制命令策略**高80%和84%**，并优于使用关节速度/位置动作的策略。
3. **动力学变化鲁棒性**：
   - 控制频率从15Hz增至30Hz：SPACE保持**94%成功率**（控制命令策略从98%降至50%），且任务完成时间从12.4s降至8.1s。
   - 负载从90g增至530g：SPACE达**92%成功率**（控制命令策略为0%），在线更新的bias（z轴）自动上升补偿重力。
   - 控制器增益变化（0.5×和1.5×）：SPACE成功率接近100%，控制命令策略分别降至25%和0%。
4. **组件必要性**：离线校准和在线更新均不可或缺；移除任一环节导致显著性能下降；使用随机轨迹进行校准优于固定形状（圆/方形）。

## 7. 优点

- **创新性**：提出笛卡尔状态增量这一通用动作表示，从根本上解耦动作与机器人动力学，且无需假设控制器结构。
- **简洁实用**：Action Adapter为线性模型+LMS更新，计算量极低，部署时无需重训练或大量数据。
- **鲁棒性广泛**：在三个层次的动力学变化（跨形态、跨硬件、运行中变化）上均表现稳健，具备实际部署价值。
- **实验设计系统**：覆盖多种真实场景，对比公平，消融充分，结果详实。
- **与现有框架兼容**：基于现代VLA模型π0.5，易于集成到其他开放策略中。

## 8. 不足与局限

- **力感知缺失**：笛卡尔状态增量无法表示力交互（相同位移可能对应不同力），不适用于需要力控的任务（如精密装配、柔软物体操作）。
- **模态转换局限**：当前Action Adapter只支持将笛卡尔状态增量转换为笛卡尔增量控制命令，对于不支持该控制模态的机器人（例如只接受关节命令）未做探索。
- **实验平台单一**：所有部署实验仅在Franka Research 3机器人上进行，且任务相对简单（拾取放置、推动），未在其他机器人（如UR5、仿人手）上验证；跨硬件实验也仅使用两台FR3。
- **校准轨迹的依赖**：需要收集随机轨迹（约1分钟），对于某些不便进行随机运动的场景可能不够便利；且轨迹形状选择影响性能（随机优于圆/方形），表明校准数据多样性很重要。
- **在线更新可能不稳定**：LMS虽简单有效，但面对非常快速或剧烈的动力学变化（如突然的大负载变化）可能无法及时收敛；论文中负载变化实验的在线更新bias变化显示适应较慢（需要若干步骤）。
- **计算资源报告不足**：未披露训练硬件和具体时长，可复现性受影响。
- **未讨论多任务联合训练**：实验中每个策略针对单个任务训练，未测试跨任务共享通用表示的效果。
