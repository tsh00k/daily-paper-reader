## 摘要
自主运行的机器人可能需要基于不再可见的证据做出决策。我们研究\emph{延迟证据}任务，其中早期线索在后续决策点之前消失，因此视觉上相似的观测可能要求不同的动作。在这些场景中，当前观测不足以作为控制的状态。我们提出轨迹路由因果证据（TRACE），一种用于视觉运动模仿策略的记忆框架。TRACE将任务相关的视觉和机器人状态证据（如物体身份、目标选择或路径依赖状态）存储在固定大小的潜记忆中，该记忆在长回合中保持有界。不同于按原始时间或手动提供的任务标签索引记忆，TRACE使用\emph{路径签名}：已执行机器人状态轨迹的紧凑且顺序敏感的特征。这些签名本身不存储视觉线索；相反，它们提供轨迹条件化的键，用于写入和检索当线索可见时存储的证据。当机器人后来遇到模糊的观测时，策略基于TRACE记忆进行条件化，以恢复缺失的上下文并选择正确的分支。TRACE通过轻量适配器附加到策略上，无需更改策略骨干、动作头或模仿目标。在具有视觉模糊分支点的真实世界长时操作任务中，TRACE在分支选择和任务成功率上优于其他基线，包括短历史和循环记忆。项目页面：https://jeong-zju.github.io/trace

## Abstract
Robots under autonomous operation may require decisions based on evidence that is no longer visible. We study \emph{delayed-evidence} tasks, where an early cue disappears before a later decision point, so visually similar observations can require different actions. In these settings, the current observation is not a sufficient state for control. We introduce TRAjectory-routed Causal Evidence (TRACE), a memory framework for visuomotor imitation policies. TRACE stores task-relevant visual and robot-state evidence, such as object identity, target choice, or route-dependent state, in a fixed-size latent memory that remains bounded over long episodes. Instead of indexing memory by raw time or manually provided task labels, TRACE uses \emph{path signatures}: compact, order-sensitive features of the executed robot-state trajectory. These signatures do not store the visual cue itself; rather, they provide trajectory-conditioned keys for writing and retrieving the evidence stored when the cue was visible. When the robot later reaches an ambiguous observation, the policy conditions on TRACE memory to recover the missing context and choose the correct branch. TRACE attaches through lightweight adapters to policies, without changing the policy backbone, action head, or imitation objective. Across real-world long-horizon manipulation tasks with visually ambiguous branch points, TRACE improves branch selection and task success over alternative baselines, including short-history and recurrent memory. ### 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：使用**路径签名**（path signature）作为轨迹条件键，将视觉和状态证据写入固定数量的潜记忆槽；在后续分支点，通过相同轨迹键检索存储的上下文，从而正确选择分支。
- **关键技术细节**：
  - **路径签名**：对机器人状态轨迹（分段线性插值）计算固定长度的、顺序敏感的摘要向量，记为 \(\xi_t = \text{Sig}^{\leq p}(S_t)\)，以及增量 \(\delta_t = \xi_t - \xi_{t-1}\)。签名对单调时间重参数化不变，但与执行顺序高度相关。
  - **记忆接口**：TRACE 维护 \(K\) 个潜槽 \(M_t \in \mathbb{R}^{K \times d}\)，每个槽存储特征。使用轨迹键 \(q_t = \phi_q([g_t, \Delta g_t])\) 计算路由权重 \(\omega_{t,k}\)（softmax），选择哪些槽写入当前证据；通过门控更新写入；再从槽中读取得到记忆读出 \(z_t^{\text{mem}}\)。
  - **策略适配**：TRACE 通过轻量适配器将记忆状态 \(R_t = (M_t, z_t^{\text{mem}}, g_t, \Delta g_t)\) 映射为策略原生条件。对回归型策略（ACT）输出记忆 token 拼接到注意力记忆；对扩散策略输出全局条件向量。不改变策略骨干、动作头或模仿损失。
- **公式与算法流程**（文字说明）：
  1. 对当前观测 \(x_t\)（多视角图像+机器人状态）编码为视觉特征 \(e_t\)。
  2. 更新路径签名 \(\xi_t\) 和增量 \(\delta_t\)，投影得到 \(g_t, \Delta g_t\)，组成轨迹键 \(q_t\)。
  3. 用 \(q_t\) 计算路由权重 \(\omega_{t,k}\)，选择历史槽。
  4. 生成写候选 \(\tilde{m}_{t,k}\)，并用门控 \(\beta_{t,k} = \omega_{t,k} \sigma(\cdot)\) 更新槽 \(m_{t,k} = (1-\beta_{t,k}) m_{t-1,k} + \beta_{t,k} \tilde{m}_{t,k}\)。
  5. 从更新后的记忆读取 \(z_t^{\text{mem}} = \sum_k \alpha_{t,k} W_V m_{t,k}\)，其中 \(\alpha\) 由当前特征和轨迹键计算。
  6. 构造 \(R_t\)，通过适配器得到条件 \(c_t\)，输入策略预测动作。

### 论文的主要结论与发现

1. TRACE 在延迟证据任务上显著优于无因果记忆的策略：回归型平均进度从25.50提升至69.23，扩散型从25.00提升至59.53，且优于最强的非TRACE基线π0.5（50.47）。
2. 增益跨策略家族（回归与扩散）可迁移，仅需更换轻量适配器。
3. TRACE 优于通用历史记忆（GRU、Transformer上下文、LRU、检索提示），因为其利用轨迹顺序来组织和检索证据。
4. 诊断实验表明TRACE确实使用了有序历史证据，而非仅依赖决策点观测：顺序保持变换下路由相似度和分支一致性高，顺序反转则显著下降。

### 优点

- **模块化设计**：TRACE 作为独立记忆模块，通过轻量适配器插入任意策略，无需修改策略骨干或模仿损失。
- **固定预算**：记忆槽数量固定，不随回合长度增长，适合长时操作。
- **轨迹路由机制**：利用路径签名的顺序敏感性，自动区分通过不同历史达到的相似视觉状态，避免信息混淆。
- **诊断工具**：路由相似度、分支一致性等指标可解释模型行为，验证记忆确实使用历史顺序。
- **真实任务验证**：在5个具有视觉歧义的物理操作任务上验证，覆盖物体身份、起源、路线等典型延迟证据情景。
