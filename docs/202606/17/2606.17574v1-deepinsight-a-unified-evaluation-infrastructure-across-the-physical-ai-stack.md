## 摘要
评估物理AI栈涵盖的操作符差异超过三个数量级——从单个基础模型解码步骤到全身控制的数千个物理刻度——在模态、奖励语义和资源特征上正交变化。现有框架无法覆盖这一范围，因此当前通过拼接独立的测试工具来评估整个栈，这些工具既不共享运行时也不共享评分，虽然保留了各段的局部有效性，但失去了诊断跨层回归所需的共同标识。我们提出DeepInsight，这是一个在单一运行时上服务于整个频谱的评估基础设施。它没有同质化这些不同的范式，而是通过三个窄抽象——任务、资源和结果——保留其异质性，每个抽象均由每个子系统共享的一个不变量实现：一个情节驱动程序、一个由每个昂贵后端（包括LLM推理和沙盒运行时）实现的资源句柄协议，以及一个追踪身份方案，每个事件都按此写入。在具身人形机器人栈的所有三层中部署此基础设施后，这一组不变量主要通过配置即可接入新基准。在成熟的对等协调器（如基础模型端）存在的场景中，它在自身散布范围内复现了已发表的参考和对等框架的读数，在单个节点上更快地运行相同基准，并且在节点间接近线性扩展。其独特的回报在于诊断能力：由于每一层都写入一个共享追踪，始于某一层并在另一层浮现的回归在该追踪上仍然可定位——这是任何分段工具联合体都无法复现的跨层收益。

## Abstract
Evaluating a Physical AI stack spans operators that differ by more than three orders of magnitude -- from a single foundation-model decoding step to thousands of physics ticks of whole-body control -- varying orthogonally in modality, reward semantics, and resource profile. No existing framework spans this range, so the stack is evaluated today by stitching together separate harnesses that share neither runtime nor scoring, preserving each segment's local validity but losing the shared identity needed to diagnose cross-layer regressions. We present DeepInsight, an evaluation infrastructure that serves this full spectrum on a single runtime. Rather than homogenize the regimes, it preserves their heterogeneity behind three narrow abstractions -- task, resource, and result -- each realized as one invariant shared by every subsystem: one episode driver, one resource-handle protocol implemented by every expensive backend (LLM inference and sandboxed runtimes alike), and one trace identity scheme under which every event is written. Deployed in production across all three layers of an embodied humanoid stack, this single set of invariants onboards new benchmarks largely by configuration. Where mature peer orchestrators exist -- at the foundation-model end -- it reproduces published references and peer-framework readings within their own spread, runs the same suites faster on a single node, and scales near-linearly across nodes. Its distinctive return is diagnostic: because every layer writes into one shared trace, a regression that begins in one layer and surfaces in another stays localizable on that trace -- a cross-layer payoff no federation of per-segment harnesses can reproduce.

## 一、核心问题与整体含义（研究动机和背景）

- **问题**：物理AI栈评估面临极大异质性——从基础模型的单步解码到全身控制的数千物理步长，操作符在模态、奖励语义、资源特征上正交变化（跨度超3个数量级）。现有评估框架（如lm-eval、OpenCompass、HELM、VLMEvalKit、Inspect AI、Isaac Lab等）均只覆盖某一特定段，无法在同一运行时上支持全栈评估。
- **痛点**：当前实践是拼接独立的测试工具（per-segment harnesses），它们不共享运行时、评分或追踪，导致**跨层回归无法诊断**——例如语义规划器的错误会改变视觉运动策略的输入分布，而政策的犹豫会影响稳定器的运行体制。分段工具保存了局部有效性但丢失了共同身份。
- **整体目标**：构建一个统一评估基础设施，能够在单一运行时上承载从基础模型推理到全身控制的全栈评估，保留各层的异质性（而非强行统一），同时使跨层回归可定位。核心价值在于**诊断能力**：所有层写入同一共享追踪，回归点可在此追踪上追溯到源头。

## 二、方法论：核心思想、关键技术细节

### 核心思想
通过三个窄抽象（task, resource, result）吸收物理AI评估频谱的异质性，每一个抽象由所有子系统共享的一个不变量实现：
- **任务抽象**：一个episode driver（reset/step接口），所有暂态状态驻留在per-episode handle上，环境本身保持无状态。
- **资源抽象**：一个resource-handle protocol（acquire/release），每个昂贵后端（LLM推理、沙盒运行时）实现此协议；后端被委托给独立的控制平面（inference plane和sandbox plane），与编排器解耦。
- **结果抽象**：一个trace identity scheme（统一的TraceRecord模式），所有事件（对话、判断、租赁、推理、轨迹）写入同一个结构化记录，支持跨源分析和因果回溯。

### 关键技术细节
1. **任务抽象**：
   - 环境只暴露reset(handle)→obs和step(handle, action)→(obs, event, termination)。所有暂态（数据集样本、对话、中间结果）存在handle上，一个环境实例可服务任意多并发episode。
   - 任务声明分为两层：suite config（定义任务是什么）和template config（定义如何运行一次）。大部分基准以配置文件而非代码形式接入。
   - 判断（scoring）与任务正交，四种scorer族（rule、model、vision、benchmark）共享同一接口，可独立替换。

2. **资源抽象**：
   - 多阶段拥有独立并发预算：GPU-bound generation、I/O-bound sandbox execution、CPU-bound scoring各有独立worker pool。
   - 两个控制平面（inference plane, sandbox plane）暴露相同的acquire/release句柄；编排器不直接管理后端资源。
   - Inference plane：支持异构引擎（vLLM, SGLang等），客户端侧负载均衡（支持KV-cache亲和性的bounded-load consistent hash），自动扩展。
   - Sandbox plane：两种模式——无状态用预预热容器池，有状态用按需OpenSandbox容器。生命周期由Kubernetes控制环管理。
   - 缩放旋钮正交：每阶段worker并发、连接池、副本数、沙盒容量独立可调。

3. **结果抽象**：
   - 所有事件写入同一个TraceRecord，具有层次化身份元组（run_id, suite_id, task_id, episode_id, epoch）和多速率位置索引（turn_id, step_id, tick），以及因果父指针（event_id, parent_event_id）。
   - 追踪是持久化制品，聚合分数从追踪计算而非替代追踪。
   - 报告（reporting）与评估（evaluation）解耦：reporters作为下游阶段消费追踪，不阻塞流水线。

### 算法流程概述（文字说明）
1. **任务规范**：加载suite config和template config，合并为执行配置快照（持久化）。
2. **环境**：环境调用reset(handle)产生初始观测；循环调用step(handle, action)直到终止。
3. **执行**：薄异步编排器通过异步队列驱动各阶段（生成→沙盒→评分），每个阶段有独立并发池。
4. **资源平面**：编排器通过acquire请求句柄，使用后release；平面（inference/sandbox）自主管理后端。
5. **评分与报告**：per-episode judge从追踪读取事件计算奖励；off-path reporters消费追踪产生报告。

## 六、主要结论与发现

1. **覆盖全栈**：DeepInsight是第一个在同一运行时上覆盖从基础模型到全身控制的物理AI评估基础设施。
2. **精度对齐**：在System 2（文本、多模态、全模态）上，DeepInsight复现参考分数和对等框架读数在其自身散布范围内，且在大多数行上最接近参考（5/8文本表2，4/9文本表3，8/15多模态表4，4/5全模态表5）。
3. **吞吐量优势**：
   - 单节点上，DeepInsight比lm-eval快1.29×，比Inspect AI快1.13×，比VLMEvalKit快1.03×，比lmms-eval快1.04×（多模态）/ 1.20×（全模态）。
   - 异步流水线（AIME-2024）带来1.41×加速，GPU占用率从54%提升至89%。
   - 阶段解耦（LiveCodeBench v6）带来3.31×加速，LLM吞吐量从731 tokens/s提升至2,385 tokens/s。
4. **水平扩展**：1→2→4节点近乎线性（2.03×, 1.97×），总加速4.00×，失败率<0.1%。
5. **跨层诊断**：全栈组合任务（车辆引导）暴露了集成漏洞：38个失败episode中，System 1占42.1%，系统边界接口失效占28.9%，System 2占18.4%，System 0占10.5%。仅凭各层独立评估无法发现这些边界失效；统一的追踪允许将回归追溯到起源层。
6. **System 0释放决策**：聚合得分最高的策略（WBC-RC-01）仍可能因局部动力学、接触姿态、上肢运动学等细粒度诊断失败而被阻止部署；追踪是决定性的制品。

## 七、优点

1. **架构优雅**：三个窄抽象有效、模块化，允许保留异质性而非强求统一；每个抽象解决一类异质性。
2. **实际部署且开源相关**：已在生产环境中部署；System 2部分与成熟框架公平对比并展现优势；System 1/0部分覆盖了现有框架完全未触及的范围。
3. **出色诊断能力**：统一追踪实现了在现有框架中不可能做到的跨层因果定位；这是论文最具新意的贡献。
4. **扩展便捷**：新基准（任意频谱段）通常只需配置文件即可接入，无需修改运行时。
5. **官方可信**：结果有多个基准、多轮运行、标准偏差报告，实验设计在同领域内属于较高标准。

## 八、不足与局限

1. **System 1/0评估深度不足**：论文明确承认“定量深度留待未来工作”，System 1仅展示覆盖和主观评估；System 0仅展示一个政策集筛选和诊断例子；缺乏像System 2那样的多基准、多策略、多轮统计分析。实验数量较少，结论更多是证明可行性。
2. **缺乏同类框架对比**：System 1和System 0没有对等框架可对比，因此Speedup或精度指标无法验证。论文声称“跨层诊断是对等框架无法实现的”，但未在同一硬件上运行任何其他框架来证明这一点。
3. **实验样本量**：全栈功能验证仅96 episodes（38个失败），统计稳定性一般；失败定位依赖于单次运行，未给出跨多轮运行的方差。
4. **未提及真实机器人部署**：所有rollout均为仿真；Sim-to-Real差距被列为未来工作。
5. **资源消耗报告**：System 1/0训练的GPU资源、时间未公开，读者无法评估经济性；System 2部分硬件虽然明确，但未报告具体推理成本或时延分解（如推理请求数量、总token数）。
6. **部分基准对齐偏差未完全解释**：例如LiveCodeBench依赖“unpublished reference sandbox/timeout”，Video-MME因字幕差异对比不公平。这些偏差确实被提及，但对齐证据的强度有所限制。
