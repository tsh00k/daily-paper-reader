## 摘要
从成对偏好中训练的奖励模型常常利用表面捷径线索而非学习真实响应质量。我们提出DynaCF，一种用于缓解奖励模型训练中捷径学习的动态重加权框架。与静态捷径启发式方法不同，DynaCF在优化过程中通过应用保持语义的反事实扰动并追踪当前模型下产生的边际偏移和偏好翻转，在线测量捷径敏感性。对捷径敏感性更高的样本在Bradley-Terry目标中被动态降低权重，从而鼓励模型更少依赖表面模式，更多依赖任务相关的偏好信号。大量实验表明，DynaCF持续提高了偏好建模的鲁棒性。

## Abstract
Reward models trained from pairwise preferences often exploit superficial shortcut cues rather than learning true response quality. We propose DynaCF, a dynamic reweighting framework for mitigating shortcut learning in reward model training. Unlike static shortcut heuristics, DynaCF measures shortcut sensitivity online during optimization by applying semantics-preserving counterfactual perturbations and tracking the resulting margin shifts and preference flips under the current model. Samples with higher shortcut sensitivity are dynamically downweighted in the Bradley-Terry objective, encouraging the model to rely less on superficial patterns and more on task-relevant preference signals. Extensive experiments show that DynaCF consistently improves robustness in preference modeling.

# DynaCF：通过动态反事实敏感性缓解奖励模型中的捷径学习 — 详细总结

## 1. 核心问题与整体含义（研究动机和背景）

在人类反馈强化学习（RLHF）中，奖励模型从成对偏好数据（chosen/rejected responses）中学习打分函数。然而，这类偏好对往往不仅包含任务相关的质量差异，还混入了大量表面线索，如响应长度、格式（Markdown、列表）、语气（自信、模板化）、冗余表述等。奖励模型容易利用这些“捷径”来区分偏好，而不是真正理解响应质量，导致泛化能力差、鲁棒性弱，尤其在困难推理和安全敏感场景中表现不佳。

现有缓解捷径学习的方法多为静态启发式（如基于固定特征过滤或增强），但捷径风险并非数据集的固有属性，而是依赖于模型当前的行为：同一个样本在不同训练阶段可能表现出不同的捷径敏感性。因此，需要一种能够在训练过程中 **动态诊断** 模型是否依赖表面线索，并据此调整训练信号的方法。

本文提出 **DynaCF**（Dynamic Counterfactual Sensitivity），旨在通过在线构建语义保持的反事实扰动，测量当前模型对表面变化的敏感性，并动态降低高敏感样本在 Bradley-Terry 损失中的权重，从而鼓励模型关注真实偏好信号。

## 2. 方法论

### 核心思想
DynaCF 在每一训练步骤，对当前批次中的每个样本计算“反事实捷径敏感性”分数，然后将其映射为样本权重，重新加权原始 Bradley-Terry 损失。关键点在于：
- 只对 **chosen response** 施加表面扰动，保持 prompt 和 rejected response 不变。
- 扰动是 **语义保持的**：仅改变格式、长度、语气、冗余度等表面特征，不改变任务核心内容。
- 敏感性分数基于 **奖励边际变化** 和 **偏好翻转** 两个指标，综合反映模型对表面线索的依赖程度。
- 权重是 **动态** 的，随模型参数更新而在线重计算，因此同一样本在不同阶段可能获得不同权重。

### 关键技术细节

#### 反事实敏感性度量
对每个样本 i，构造 K 个有效反事实变体 \( y_{i,k}^{\text{cf}} \)，保持 prompt \(x_i\) 和 rejected \(y_i^-\) 不变。计算原始边际 \(m_i = R(x_i,y_i^+) - R(x_i,y_i^-)\) 和反事实边际 \(m_{i,k}^{\text{cf}} = R(x_i,y_{i,k}^{\text{cf}}) - R(x_i,y_i^-)\)。定义：
- 边际变化 \(\Delta_{i,k} = m_i - m_{i,k}^{\text{cf}}\)
- 偏好翻转指示器 \(F_{i,k} = \mathbb{I}[m_{i,k}^{\text{cf}} < 0]\)

敏感性分数为：
\[
s_{i}^{\text{cf}} = \frac{1}{K_i} \sum_k \Delta_{i,k} + \mu_{\text{flip}} \cdot \frac{1}{K_i} \sum_k F_{i,k}
\]
其中 \(\mu_{\text{flip}}\) 控制翻转惩罚权重。

#### 动态重加权
将敏感性分数转换为样本权重：
\[
w_i(t) = \max\left( w_{\min},\ 1 - \gamma \cdot s_{i}^{\text{cf}}(t) \right)
\]
其中 \(\gamma\) 是降权强度，\(w_{\min}=0.1\) 防止样本被完全丢弃。

最终损失为：
\[
\mathcal{L}_{\text{DynaCF}}(t) = \frac{1}{|B_t|} \sum_{i \in B_t} w_i(t) \left[ -\log \sigma(m_i(t)) \right]
\]

#### 反事实构造
采用 **领域感知的规则**（非自由 LLM 编辑）：
- **通用领域**（chat, safety）：移除 Markdown 格式、缩短冗余、中性化语气、去除模板化表达。
- **数学领域**：保守处理，保留数字和方程，仅简化非核心说明。
- **代码领域**：保留代码块和逻辑，简化外围文字和注释。

每个反事实需通过有效性检查（语义重叠阈值、数字保留等），未通过则 \(K_i=0\)，该样本保持原权重。

#### 训练流程
- 先进行 5% 步骤的 **预热**（标准 BT 损失），之后启用动态重加权。
- 敏感性分数在每一步（或每一定频率）使用当前模型重新计算。

## 6. 主要结论与发现

1. **困难样本受益更大**：在 RM-Bench Hard 分上，DynaCF 对 Qwen3-4B 提升 +8.7 个点（41.3% → 50.0%），对 Qwen3-8B 提升 +4.6 个点（49.8% → 54.4%），同时保持 Easy 性能几乎不变。
2. **安全评估显著提升**：在 RM-Bench Safety 上，Qwen3-4B 提升 +8.2 个点，RewardBench 和 RewardBench2 安全分数也分别提升 +4.0 和 +6.1。
3. **动态性至关重要**：静态敏感度（训练前计算）反而降低性能，说明模型尚未形成决策边界；动态重计算越频繁，效果越好。
4. **诊断式设计优于训练式**：仅将反事实作为探针（不额外训练）效果更好，直接加入反事实对作为训练样本反而有害。
5. **预热必要**：立即启用重加权效果不佳，先进行 5% 步骤的 BT 训练后再启动最优。
6. **域感知的反事实构造**：针对数学、代码等领域的定制规则比通用规则更有效。

## 7. 优点

- **自适应性与动态性**：不依赖静态启发式或预定义特征，实时根据模型当前状态调整权重大小，更精准地识别真正有问题的样本。
- **语义保持的反事实构造**：通过规则而非 LLM 生成，可控性强，且通过重叠度、数字保留等有效性检查，保证不破坏任务内容。
- **无需删除样本或改变数据分布**：仅通过软权重调整，保留了有用监督信息，减少信息损失。
- **简单有效**：在现有 Bradley-Terry 框架上轻量级修改，计算开销可控（约 12%）。
- **广泛消融**：对每个设计选择（动态频率、预热、超参数、反事实规则）都进行了系统验证，结论可靠。

## 8. 不足与局限

- **反事实质量依赖**：尽管有规则和有效性检查，仍可能偶尔改变有用信息或未能完全消除目标捷径。未来需更鲁棒的生成和验证。
- **仅限文本模态**：未在多模态奖励模型、过程奖励模型或交互式训练环境中测试，通用性待验证。
- **仅对 chosen response 探测**：rejected response 中也可能存在表面线索（如过短、格式不合理），仅扰动 chosen 可能不完整。
- **未探索与模型容量、训练动力学的关系**：更强模型可能表现出不同敏感性模式，且捷径敏感性在训练中如何变化尚不明确。
- **潜在偏差风险**：反事实构造的规则可能引入新的偏差（如偏向朴素表达），需谨慎设计并验证。
- **实验场景有限**：仅涉及文本偏好学习，未考虑更广泛的 RLHF 流程（如与 PPO 结合），也未评估对下游策略优化的影响。
