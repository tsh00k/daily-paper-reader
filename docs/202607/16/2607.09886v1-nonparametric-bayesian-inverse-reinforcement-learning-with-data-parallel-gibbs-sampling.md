## 摘要
逆强化学习从专家演示中恢复奖励函数，但标准公式假设所有演示来自单个专家。当演示来自具有不同偏好的多个专家时，参数化方法会恢复出一个平均奖励，无法很好地拟合任何单个专家。我们采用非参数贝叶斯逆强化学习，在奖励函数上使用狄利克雷过程先验，使得潜在奖励类型的数量可以与奖励本身联合推断。推理使用折叠吉布斯采样器，结合中国餐馆过程更新进行聚类分配，以及Metropolis-Hastings更新奖励权重，并以软值迭代作为内部规划例程。我们在10x10的ObjectWorld网格上评估，具有两个和三个真实奖励类型。串行采样器恢复K=2时调整兰德指数为1.000，显著优于最大熵IRL基线（ARI=0.000）。扩展到K=3时，采样器在所有运行中正确识别了聚类数量；分配ARI为0.48-0.58反映了专家类型之间的行为重叠，这种重叠在各个网格实例中持续存在，表明在ObjectWorld上进行可靠的K=3评估需要受控的对象放置而非随机播种。我们进一步使用Ray在HPC硬件上跨CPU核心并行化采样器，在8个工作节点时达到4.79倍的峰值加速，并描述了由状态聚合期间使用的共识合并启发式方法导致的吞吐量与准确性之间的权衡。代码和容器化环境可在https://github.com/dasashreeya/np_bayes_irl获取。

## Abstract
Inverse Reinforcement Learning recovers reward functions from expert demonstrations, but standard formulations assume that all demonstrations come from a single expert. When demonstrations are pooled from multiple experts with distinct preferences, parametric methods recover an averaged reward that fits no individual expert well. We implement Nonparametric Bayesian Inverse Reinforcement Learning with a Dirichlet Process prior over reward functions, allowing the number of latent reward types to be inferred jointly with the rewards themselves. Inference uses a collapsed Gibbs sampler combining a Chinese Restaurant Process update for cluster assignments with a Metropolis-Hastings update for reward weights, and soft value iteration as the inner planning routine. We evaluate on a 10x10 ObjectWorld grid with two and three ground-truth reward types. The serial sampler recovers K=2 with Adjusted Rand Index of 1.000, substantially outperforming a Maximum Entropy IRL baseline (ARI=0.000). Extension to K=3 shows that the sampler correctly identifies the number of clusters in all runs; assignment ARI of 0.48-0.58 reflects behavioral overlap between expert types that persists across grid instantiations, revealing that reliable K=3 evaluation on ObjectWorld requires controlled object placement rather than random seeding. We further parallelize the sampler across CPU cores using Ray on HPC hardware, achieving a peak speedup of 4.79x at 8 workers, and characterize a throughput-versus-accuracy tradeoff arising from the consensus merge heuristic used during state aggregation. Code and a containerized environment are available at https://github.com/dasashreeya/np_bayes_irl.

# 论文总结：Nonparametric Bayesian Inverse Reinforcement Learning with Data-Parallel Gibbs Sampling

## 1. 核心问题与整体含义（研究背景与动机）

- **研究背景**：标准逆强化学习（IRL）假设所有演示来自单一专家，并据此恢复一个奖励函数。但在实际场景中，演示数据往往来自多个具有不同偏好的专家（如不同驾驶风格）。
- **核心问题**：如果混合多专家数据，参数化IRL方法（如最大熵IRL）只能恢复一个平均奖励，无法拟合任何单个专家的偏好。此外，专家类型的数量通常是未知的。
- **研究动机**：需要一种能够自动推断潜在奖励类型数目（聚类数）并同时恢复每个类型奖励函数的方法。非参数贝叶斯方法（狄利克雷过程）天然适合此类问题，但计算成本高，需要并行化加速。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：使用狄利克雷过程（DP）先验对奖励权重分布建模，允许聚类数目随数据自适应增长；通过折叠吉布斯采样（collapsed Gibbs sampler）联合推断聚类分配与奖励权重。
- **生成模型**：
  - \( G \sim DP(\alpha, G_0) \)，其中 \( G_0 = N(0, I) \) 为基础分布，\(\alpha\) 为浓度参数。
  - 每个轨迹 \( \tau_i \) 对应的权重 \( w_i \sim G \)。
  - 轨迹条件分布采用 Boltzmann 理性专家模型：\( p(\tau | w) = \prod_t \pi_{\beta}^{w}(a_t | s_t) \)，其中策略由软值迭代计算的 Q 函数导出。
- **推理算法**（每个吉布斯扫描包含两步）：
  1. **聚类分配更新**：使用中国餐馆过程（CRP）公式，计算轨迹 \( \tau_i \) 分配到现有类 \(k\) 的概率（正比于 \( n_{-i,k} \cdot p(\tau_i | w_k) \)）或新类的概率（正比于 \( \alpha \cdot p(\tau_i | w_{\text{new}}) \)）。
  2. **权重更新**：对每个类 \(k\)，使用 Metropolis-Hastings 算法提出新权重 \( w'_k = w_k + \epsilon, \epsilon \sim N(0, \sigma^2 I) \)，并按接受概率更新。所有似然计算使用 log-space 防止下溢。
- **并行化**：将轨迹划分到多个 Ray 工作节点，每个节点独立执行一次吉布斯扫描，然后通过共识合并步骤（元素级比较，容差 \(10^{-6}\)）合并来自不同节点的状态。环境张量预先放入 Ray 分布式对象存储。
- **实现**：基于 JAX 实现约 2100 行 Python 代码，内层 Boltzmann log-policy 计算使用 JIT 编译；外循环保持 Python 灵活性。

## 3. 实验设计

- **环境**：ObjectWorld 基准，10×10 网格，100 个状态，4 个动作。特征向量 \(\phi(s) \in \mathbb{R}^{16}\) 是二值的，编码每个单元格最近和第二近物体的颜色（8 种颜色，2 个距离带）。
- **专家类型设置**：
  - **K=2**：红色爱好者（偏好红色物体，厌恶蓝色）+ 蓝色爱好者（偏好蓝色，厌恶红色）。每种类型 10 条轨迹，共 20 条，每条长度 20 步。
  - **K=3**：额外加入黑色爱好者（偏好黑色，厌恶绿色）。每种类型 10 条轨迹，共 30 条。注意默认网格种子下蓝色和黄色物体缺失，导致蓝色爱好者简化为“避免红色”信号。
- **对比方法**：最大熵 IRL（MaxEnt IRL），使用特征匹配梯度下降，200 次迭代，学习率 0.01。由于 MaxEnt 只拟合单一奖励，对异构数据 ARI 必然为 0。
- **评估指标**：
  - 调整兰德指数（ARI）：聚类恢复准确性（0=随机，1=完美）。
  - \(\ell_2\) 权重误差：每个真实权重与最近恢复权重的归一化欧氏距离。
  - 墙钟时间。
- **并行化研究**：在固定 100 次扫描下，使用 1, 2, 4, 8, 16 个工作节点测量加速比。

## 5. 实验数量与充分性

- **主要实验组**：
  - K=2 场景：串行与非并行对比，与 MaxEnt 基线对比。
  - K=3 场景：评估聚类数目恢复与分配准确率，并测试了不同网格种子（seed=42, seed=19）对结果的影响。
  - 超参数消融实验：对浓度参数 \(\alpha\)（1.0, 3.0, 5.0, 7.0, 10.0）和 MH 提案尺度 \(\sigma\)（0.001, 0.01, 0.05, 0.1）进行 300 次扫描下的 ARI、\(\ell_2\) 误差和耗时比较。

## 6. 主要结论与发现

1. **K=2 完美恢复**：串行非参数贝叶斯 IRL 在 ObjectWorld K=2 场景下，经过约 250 次扫描收敛，500 次扫描后达到 ARI=1.000，\(\ell_2\) 权重误差 1.355。基线 MaxEnt IRL 的 ARI=0.000。
2. **K=3 受限于环境可区分性**：采样器正确推断出 K=3，但分配 ARI 仅为 0.48–0.58。原因在于默认网格中蓝色物体缺失导致蓝色爱好者与黑色爱好者轨迹高度重叠，行为可区分性不足。只有 1/20 的随机种子生成包含所有三种颜色的网格，但仍无法实现完美分配，因为密度差异导致持续合并。
3. **并行加速**：8 个 worker 时达到 4.79 倍峰值加速（100 次扫描），2 个 worker 出现超线性加速（2.76 倍，归因于 JIT 编译摊销）。超过 8 个 worker 后，工作负载太小导致协调开销占主导，加速比下降。
4. **吞吐量与精度权衡**：随着 worker 数增加，ARI 急剧下降（从 0.905 降至 0.035），但 \(\ell_2\) 权重误差保持稳定。说明精度损失源于合并步骤中独立 MH 更新导致的权重漂移，而非奖励质量下降。可通过集中式权重更新修复。
5. **超参数敏感性**：\(\alpha\) 在 1.0–5.0 范围内收敛到同一中间模态（K=3, ARI=0.458），\(\alpha=7.0\) 得到最佳 ARI=0.667 但 K=4（过分割）；\(\sigma=0.01\) 表现稳健，\(\sigma=0.001\) 因混合慢而变差。最终选择 \(\alpha=5.0, \sigma=0.01\)。
6. **关键洞见**：合成 IRL 基准设计时，特征可区分性不能仅凭奖励规范假设，必须针对具体环境实例验证。

## 8. 不足与局限

- **实验覆盖有限**：仅在一个小规模网格环境（ObjectWorld 10×10）上评估，未测试连续状态空间、更高维特征或真实驾驶/机器人演示数据。

- **缺乏多次运行统计**：所有报告结果似乎基于单次运行（seed=0），未报告多次随机初始化的均值和方差，降低了结论可靠性。
